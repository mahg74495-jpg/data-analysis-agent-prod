# 版本开发日志

## v1.0
- 项目正式发布第一个稳定版本
- 支持基础的数据上传、查看、分析和可视化
- 支持 Excel、CSV、数据库等常见数据源
- 可通过对话方式完成常见数据分析任务
- 提供基础的图表生成与结果导出能力

---

## v2.0
**2026年5月11日**

本次升级主要围绕 **更强的分析能力、更一致的展示体验、更好的导出能力** 展开。

### 1. 日志系统升级
- 新增统一日志系统
- 自动记录运行过程、工具调用、模型请求与异常信息
- 日志按天轮转，保留 30 天，便于排查问题

### 2. 配色方案统一
- 修复图表和 PPT 配色不一致的问题
- 用户切换品牌配色后，图表和报告会同时生效
- 默认配色为 McKinsey，也支持 BCG 等方案

### 3. Agent 架构重构
- 将原本单文件的 Agent 拆分为多个模块
- 提升代码可维护性和扩展性
- 后续新增功能会更容易接入

### 4. 国际化支持
- 新增中英双语切换
- 前端界面可一键切换中文 / English
- 页面文案、提示、命令说明等已统一适配

### 5. 斜杠命令优化
- 将原来的通用分析命令拆分为更明确的命令
- 如：`/decile`、`/tree`、`/kmeans`
- 用户更容易理解每个命令的用途

### 6. 导出功能增强
- 新增 Excel 和 Word 报告导出
- 支持将分析结果直接生成文件并下载
- `/export` 和 `/report` 命令可快速完成导出任务

### 7. 数据清洗功能增强
- 新增数据概览、缺失值处理、缩尾、截尾等清洗能力
- 支持更灵活的数据预处理流程
- 清洗后数据可直接用于后续分析

### 8. 数据预览升级
- 原来的数据结构查看改为表格预览
- 更接近 Excel 的查看体验
- 可快速浏览各个表、字段和样本数据

### 9. Token 用量追踪
- 新增上下文窗口与输出 token 统计
- 可在界面中查看当前 token 使用情况
- 便于控制模型成本和上下文长度

### 10. 思考模式展示
- 支持展示模型的推理过程
- 在需要时可查看模型思考链
- 适合复杂分析任务的调试与理解

### 11. 前端体验优化
- 页面结构进一步拆分，维护更清晰
- 样式与脚本独立管理
- 界面交互更稳定，响应更流畅

## v3.0
**2026年5月18日**

本次升级聚焦于 **外部生态集成与业务知识增强**，大幅拓展了 Agent 的数据接入能力和领域适应能力。

### 1. MCP 工具调用能力
- 新增 MCP（Model Context Protocol）协议支持，Agent 可动态调用外部 MCP 工具
- 支持调用计算器、代码执行器、第三方 API 封装工具等，扩展分析能力边界
- 通过标准化协议，可接入符合 MCP 规范的任意工具生态
- 工具调用过程自动记录至日志，便于调试和审计

### 2. 业务知识库集成
- 新增业务数据库功能，支持导入企业内部的业务资料、产品文档、行业报告等
- 自动对资料进行向量化处理，构建可检索的知识库
- Agent 在执行分析时会自动检索相关知识，提升对特殊业务场景的理解和洞察能力
- 支持多种格式：Word、Excel等，满足常见文档导入需求

### 3. 数据源扩展：Google Sheets 与自定义 API
- **Google Sheets API 集成**：支持直接读取 Google Sheets 中的数据作为分析源
- **自定义数据库 API 接口**：提供通用 API 适配器
- 所有外部数据源接入后均可使用数据清洗、预览、分析等全套功能

## v4.0
**2026年5月29日**

本次升级聚焦于 **前端体验全面重构、图表稳定性提升与工程质量强化**。

### 1. 用户界面优化
- 侧边栏重构为三段式信息架构：状态区、操作区、历史区
- 新增**模型连接测试**指示点（选择模型后自动测试，也可在设置中手动触发）
- Agent 输出气泡改为"报告风格"：左侧品牌色竖线 + 底纹背景，与图表框视觉统一
- 支持一键切换暗黑模式
- 将 `agent_chat.css` 拆分为 `tokens`、`base`、`chat`、`modals`、`kb` 五个子模块
- 增加操作指南，包含常见问题说明

### 2. 增强稳定性
- 将图表依赖本地化，消除外部 CDN 依赖
- 修复加载历史对话时强制覆盖当前已选模型的问题：仅在用户尚未选择模型时才从历史恢复

### 3. 批量数据处理能力提升
- 改变原数据源接入后转换逻辑，从SQL Slight转换为duckdb，万行数据秒处理

### 4. 新增时间序列分析模块
- 支持Prophet 、SARIMA、ARIMA、VAR、GRU模型
